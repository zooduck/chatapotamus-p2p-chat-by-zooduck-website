By default, the <chatapotamus-p2p-chat-by-zooduck> element can be hosted on any secure website.

Once a connection is established by the element, all communications are made directly between two peer devices, without any middleware, and messages are stored on your local machine only (using window.localStorage).

However, you should be wary of using this element on just any site, because it is impossible to know whether or not your message and/or video data will be captured by that site.

It is therefore recommended that you host the element on your own site if you want to guarantee your privacy.

Getting started

Download the chatapotamus-p2p-chat-by-zooduck.zip to your machine. (This is a .zip archive of the zooduck/chatapotamus-p2p-chat-by-zooduck repository).

Next, unzip the archive (we suggest using 7Zip to avoid "path too long" errors on Windows) and copy the modules folder to your project, making sure to place it in the root directory where files are served.

For example, if your project's index.html is located at my-project/public/index.html the modules path should be my-project/public/modules.

If you cannot place the modules folder at the root, please consult the "Fixing assets path issues" section in the README.md file.

Add the following script tag to the document head: 

<script src="path/to/chatapotamus-p2p-chat-by-zooduck.distributable/index.module.js" type="module"></script>

Then, add the element to your page: 

<chatapotamus-p2p-chat-by-zooduck api-docs></chatapotamus-p2p-chat-by-zooduck>

Note: The api-docs attribute is optional.

Using the <chatapotamus-p2p-chat-by-zooduck> element

Standalone Mode

This is the default.

In this mode the component provides a user interface for creating unique handles and generating session description offers and answers to be shared between two peers.

How that information is shared between the two peers is entirely up to them (email, messenger, etc).

This makes the connection incredibly secure, since the unique session description offers and answers required to establish the connection are known only to the two peers.

Production Mode

To enable production mode, set the production-mode attribute: 

<chatapotamus-p2p-chat-by-zooduck production-mode></chatapotamus-p2p-chat-by-zooduck>

In this mode the user interface is limited to post-connection actions only (such as messaging, VoIP, file sending and screen sharing).

All connection related logic must be handled using the methods and events provided by the component.

This mode is therefore intended for developers wishing to incorporate the component into their application and handle the exchange of session description offers and answers between component pairs using a central server.

More details can be found by clicking the Docs CTA1 in the component header.

1To enable the Docs CTA, set the api-docs attribute on the component. Setting the production-mode attribute permanently disables this feature.
